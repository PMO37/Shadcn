<Steps>

### Create project

If you haven't already created a project, you can do so with the following command.
Also be sure you are using the correct versions of node and yarn.

```bash
yarn create redwood-app [APP_NAME]
```

### Add Tailwind

Redwood has a built-in utility for setting up tailwind.

```bash
yarn rw setup ui tailwindcss
```

### Add dependencies

Redwood uses yarn workspaces, so install the necessary dependencies for the web with the following command:

```bash
yarn workspace web add tailwindcss-animate class-variance-authority clsx tailwind-merge
```

### Add icon library

If you're using the `default` style, install `lucide-react`:

```bash
yarn workspace web add lucide-react
```

If you're using the `new-york` style, install `@radix-ui/react-icons`:

```bash
yarn workspace web add @radix-ui/react-icons
```

### Add a cn helper

I use a `cn` helper to make it easier to conditionally add Tailwind CSS classes. Here's how I define it in `web/src/lib/utils.ts`:

```ts title="web/src/lib/utils.ts"
import { clsx, type ClassValue } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

### Create components.json file

Typically this would be set up by the shadcn CLI `init` command, but copy it manually from here:

```json title="web/components.json"
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "aliases": {
    "components": "src/components",
    "ui": "src/components/ui",
    "utils": "src/lib/utils"
  },
  "style": "default",
  "tailwind": {
    "baseColor": "neutral",
    "config": "./config/tailwind.config.js",
    "css": "./src/index.css",
    "cssVariables": true,
    "prefix": ""
  },
  "rsc": false,
  "tsx": true
}
```

### (optional) Create npm alias for the shad CLI

In order to add components via the CLI, you will need to run it inside the `web/` directory, so:

```bash
cd web
npx shadcn-ui@latest add button
```

Since most work is done at the root directory of a redwood project, consider adding a new
npm script which does this for you:

```diff title="package.json"
 {
   "private": true,
+  "scripts": {
+    "shad": "cd web && npx shadcn-ui@latest add"
+  },
   "workspaces": {
     "packages": [
       "api",
       "web"
     ]
   },
```

Now, you'll be able to add shad components from the root folder:

```bash
yarn shad button tooltip calendar
```

### That's it

You can now start adding components to your project.
